# 카메라 시야각(FOV) 설계 가이드 — Phase 3/4

## 개요

산업용 감시 로봇암: RGB 카메라 + MLX90640 열화상 모듈(적층 배치)의 시야각 선택 기준.
두 센서 **시야각 매칭**, **추적 정확도**, **감시 커버리지**의 균형을 맞추는 방법.

---

## 현재 하드웨어 스펙

### RGB 카메라 (OV5647, Raspberry Pi CSI)
- **기본 (70° 제조사 기준) → 와이드 모듈 사용 중**
- 실제 측정(OpenCV 카메라 캘리브레이션 후): ~100–110° (대각선)
- 해상도: 640×480 (감시용 기본, 추적엔 충분)
- 특성: **왜곡 있음(배럴)** → 퓨전 affine으로 보정, 완벽한 건 undistort 필요

### MLX90640 열화상
- **시야각: 55° (공식), 일부 모델 110°** — 사양서 확인 필수
- 해상도: 24×32 픽셀 (매우 저해상도)
- 온도 범위: -40~85°C (감시용 기본 0~60°C)
- 특성: **고정된 시야각**, 줌/팬 불가 → 카메라 팬/틸트로 열점 추적

---

## 시야각 선택 기준

### 1. 감시 거리 vs 시야각 (관계식)

```
FOV (도) ≈ 2 × arctan(커버 폭 / 2 / 거리)

예시 (거리 2m, 폭 3m):
  FOV ≈ 2 × arctan(1.5/2) ≈ 81°

예시 (거리 1m, 폭 3m):
  FOV ≈ 2 × arctan(1.5/1) ≈ 150° (필요)
```

**추천:** 감시 거리 **1~3m**, 커버 폭 **2~4m** → **90~110° 이상 필요**

### 2. 추적 정확도 vs 시야각

| FOV | 특성 | 추적 정확도 | 용도 |
|---|---|---|---|
| ~60° (표준) | 좁음 | 높음 | 원거리 사람 추적, 얼굴인식 |
| 90~110° (와이드) | **현재** | 중상 | 근거리 감시, 팬/틸트 보정 가능 |
| >120° (울트라) | 매우 넓음 | 낮음 (가장자리 왜곡 심함) | 전체 방 커버, 워프 심함 |

**현재 와이드(~105°) → 추적은 중상 수준, 정확도 위해 팬/틸트 adjust 필수**

### 3. 두 센서 매칭 필수

```
RGB FOV ~105° (와이드)
MLX FOV  55° (기본)

⚠️ 문제: RGB에는 보이는데 열화상에 안 보임 (가장자리)
  → 추적 대상이 시야에서 벗어나도 RGB는 계속 봄
  → Fusion 오버레이가 한쪽에 몰림

✓ 해결:
  (a) 열화상 모듈을 110° 모델로 교체 (비용)
  (b) RGB 카메라 FOV를 열화상에 맞춰 사용 (현 설계)
  (c) 팬/틸트로 보정 (현재 방식 — 권장)
```

**현재 설계: (c) 팬/틸트 보정** — FSM이 열 중심 또는 사람 중심으로 암을 돌려 항상 **두 센서 중앙이 같은 점**을 보게 함.

## 실무 적용 가이드

### 설치 높이 & 거리

```
전형적인 감시 시나리오:
  - 설치 높이: 1.5~2.5m (천장 또는 높은 브래킷)
  - 감시 거리: 2~5m (사무실/복도/공장)
  - 커버 폭: 2~4m

이 경우:
  - 필요 FOV: 90~110° ✓ (현 와이드 충분)
  - 열화상 55° FOV는 **대략 2~3m에서 1~1.5m 폭 커버** ← 제약
    (즉, 화면 중앙 근처만 정확)
  - 가장자리는 affine으로 보정, 완벽하지 않음(배럴 왜곡)
```

### 추적 성능 기준

```
현 하드웨어 (RGB ~105°, MLX 55°, 팬/틸트 보정):
  ✓ 중앙 ~ 화면 75% 범위: 사람/열점 추적 ~15fps, 정확도 양호
  ⚠️ 가장자리 25%: RGB 왜곡 + 열화상 FOV 밖 → 정확도 저하
  → 팬/틸트가 빨리 움직여 대상을 중앙으로 가져와야 함
```

## 권장 설정 (현 하드웨어)

### 카메라 위치
- **높이:** 2.0~2.5m (사람 눈높이보다 위)
- **거리:** 2~3m (팬/틸트 1m 움직임 범위 = 추적 충분)
- **타겟 커버:** 2~3m 폭 (가정: 좁은 복도/출입문)

### 팬/틸트 설정
```yaml
# surveillance_config.yaml patrol 섹션
sweep_min_deg: 40      # J0 base 최소 각도 (왼쪽)
sweep_max_deg: 140     # J0 base 최대 각도 (오른쪽)
sweep_speed_dps: 12    # 느린 스윕 (초당 12도)
```
**해석:** 100° 범위 스윕 = 설치 위치로부터 좌우 ~2m 커버 (거리 2~3m에서)

---

## 현 설정의 한계 & 개선책

### 한계
1. **열화상 55° FOV** — 가장자리 추적 약함
2. **배럴 왜곡** — affine(5 DOF)으로 완벽히 못 잡음
3. **저해상도 열화상(24×32)** — 세밀한 위치 추적 어려움

### 개선 (선택사항)

| 개선 | 비용 | 효과 | 기간 |
|---|---|---|---|
| 열화상 모듈 → 110° FOV | 중간 | FOV 매칭 → 정확도 ↑ | 하드웨어 교체 |
| 카메라 undistort(체스보드) | 낮음 | 배럴 왜곡 제거 | 2h 캘리브 |
| 터렛 추가(독립 고정 열카) | 높음 | 별도 감시 패턴 | 아키텍처 변경 |

## 마이그레이션 체크리스트

설치 후 검증:

- [ ] **FOV 검사:** 카메라 시야 내에 열화상 시야 포함 여부 (가장자리 ±10% 범위)
- [ ] **추적 테스트:** 움직이는 대상이 화면 중앙 ±30%에서 부드럽게 추적되는가
- [ ] **정합 오버레이:** 오버레이에서 열점이 RGB 물체 위에 있는가 (±5% 오프셋 허용)
- [ ] **감시 커버리지:** 전체 경계를 팬/틸트로 스캔했을 때 블라인드스팟 없는가
- [ ] **추적 게인 조정:** 급격한 대상 움직임에 진동/지연 없는가 (Kp/deadzone 튜닝)

## 디버깅: 시야각 문제 진단

### 증상: 열점이 RGB에 보이지 않음
```
진단:
  1. 열화상 모듈 FOV 스펙 재확인 (55° vs 110°)
  2. 설치 거리 재측정 (터무니없이 멀면 작은 FOV로도 안 보임)
  3. 모듈 정렬 (물리적으로 기울어짐?)
  
해결:
  - 팬/틸트 범위 확대 (sweep_min/max 조정)
  - 또는 거리 단축 / 터렛 재배치
```

### 증상: 추적 떨림 (oscillation)
```
진단:
  1. Kp 너무 높음 (과응답)
  2. 열화상 저해상도(24×32)로 제어 신호 너무 세밀함
  
해결:
  - surveillance_config.yaml에서 tracking.kp_pan ↓, tracking.kp_tilt ↓
  - deadzone ↑ (기본 0.06 → 0.08~0.10)
  - max_nudge ↓ (기본 4.0 → 2.0~3.0)
```

### 증상: 추적 지연 (lag)
```
진단:
  1. detector process_every_n 너무 높음 (프레임 스킵 많음)
  2. 팬/틸트 명령이 ESP32에 도달하지 않음 (WiFi/MQTT 연결 확인)
  
해결:
  - surveillance_config.yaml perception.process_every_n = 1 (매 프레임 탐지)
  - 또는 detector 백엔드를 motion(빠름)으로 변경
  - mosquitto_sub -t 'arm/#' -v 로 MQTT 메시지 흐름 확인
```
